## What is CogniBoiler?

CogniBoiler is a **digital twin platform** that simulates the complete thermodynamic cycle of a modern 300 MW gas-fired steam power plant вҖ” boiler, turbine, and virtual PLC вҖ” enriched with an AI layer for anomaly detection, efficiency optimization, and predictive maintenance.

The project is designed from the ground up to reflect real industrial software: it follows IEC standards, uses the same protocols found on actual plant floors (OPC UA, MQTT), and is built to be deployable on real infrastructure via Kubernetes.

This is not a toy simulation. The physics model is derived from real thermodynamic equations. The AI models train on that physics вҖ” not random noise. The security model follows NERC CIP and IEC 62443 conventions. Every architectural decision maps to something you would find in production at a utility company.

> **Target markets:** United States В· Uruguay В· Argentina

## AI Capabilities

Three independent neural network models run on a continuous inference schedule:

**Anomaly Detector** вҖ” LSTM Autoencoder trained exclusively on normal operating data. Detects deviations via reconstruction error. Targets: burner fouling, steam leaks, turbine blade wear.

**Efficiency Advisor** вҖ” Temporal Transformer that takes 30 minutes of plant history and recommends optimal setpoints for the next 15 minutes. Includes a physical constraint layer to ensure all recommendations are thermodynamically feasible.

**Predictive Maintenance** вҖ” GRU network computing a health score (0вҖ“100) for each component, combined with Weibull Survival Analysis to estimate time-to-failure probability. Output: prioritized maintenance schedule with confidence intervals.

All three models train on **synthetic data generated by the physics engine itself** вҖ” making the project fully self-contained and reproducible without access to real plant data.

## Physical Model

The boiler is described by a system of ODEs solved numerically (Runge-Kutta 4/5):

```
Thermal balance:    dU/dt  = Q_fuel - Q_steam - Q_loss
Steam pressure:     dP/dt  = f(T, m_water, V_drum)
Drum water level:   dh/dt  = (m_feed - m_steam) / (ПҒ В· A)
Flue gas temp:      dT_g/dt = f(Q_fuel, m_air, О·_combustion)
Turbine power:      W_elec = О·_turbine В· m_steam В· (h_in - h_out)
```

**Nominal parameters** (300 MW class gas boiler):

| Parameter         | Value          |
| ----------------- | -------------- |
| Steam pressure    | 100 вҖ“ 180 bar  |
| Steam temperature | 540 вҖ“ 565 В°C   |
| Steam flow        | 500 вҖ“ 1000 t/h |
| Electrical output | 100 вҖ“ 300 MW   |
| Boiler efficiency | 88 вҖ“ 93 %      |

Control is implemented as a **cascade PID system** вҖ” the industrial standard for boiler control:

- `PID_1` Master: power setpoint -> pressure setpoint
- `PID_2` Slave: pressure setpoint -> fuel valve position
- `PID_3` Independent: water level -> feedwater valve
- `PID_4` Independent: steam temperature -> desuperheater spray

## Security Model

Security is implemented at every layer independently вҖ” compromise of one layer does not grant access to the next.

```
Transport:   TLS 1.3 (all HTTP)  В·  mTLS (MQTT, OPC UA, gRPC)
Identity:    JWT RS256 В· 15-min access tokens В· 7-day refresh rotation
Passwords:   bcrypt (cost 12) вҖ” plaintext never stored
Data at rest: AES-256-GCM for sensitive fields
Audit log:   INSERT-only PostgreSQL table вҖ” physically immutable
Network:     Kubernetes NetworkPolicy вҖ” zero-trust between namespaces
Scanning:    Trivy (images) В· Bandit (Python source)
```

RBAC roles: `viewer` В· `operator` В· `engineer` В· `admin`
